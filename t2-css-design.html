The four position elements.

relative, fixed, absolute & static

Sunday, August 9th

The position property specifies the type of positioning method used for an element(static,relative,absolute or fixed),elements are then positioned using bottom, top, left or right. Altough none of this were work until you set the position first , they also work differently depending on the position value.

     Example{
     position: fixed;
     position: absolute;
     position: static;
     position: relative;
   }

One of them is the position Static and this are position static by default, this is not affected by the top, right, left and bottom properties. An element with position: Static is not position in any special way, is always position according to the normal flow of the page.

The posistion relative, is positioned relative to its normal position. Setting to the top , right, bottom and left elements will cause it to be adjusted away from its normal position.

      Example
      .hello {
      position: relative;
      left: 20px;
    }
    The elements with the hello class  will move towards the left from the original position that you had before the css.

Position fixed , is position relative to the viewport which means it always stays at the same place even someone scrolled down the page. The top ,right,bottom and left are used to position the element. A fixed element does not leave a gap in the page where it would normally have been located.

An element with position absolute is positioned relative to the nearest position ancestor(is similar as the position fixed) However an absolute position element has no positioned ancestors, it moves around with the page and it moves along while scrolling the page.
